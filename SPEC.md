# 校园拼单助理 - 产品规格说明书

## 1. 项目概述

### 1.1 项目名称
校园拼单助理 (Campus Group-Buy Assistant)

### 1.2 项目类型
轻量级B/S架构网页应用，面向高校师生的拼单信息发布与撮合平台

### 1.3 核心功能概述
一个信息发布与社交连接器，用户可发布/浏览/加入拼单，通过获取联系方式后完成线下交易

### 1.4 目标用户
在校大学生及教职工，熟悉互联网应用，使用场景碎片化

---

## 2. 技术方案

### 2.1 前端技术
- HTML5 + CSS3 + JavaScript (原生)
- CSS变量 + Flexbox/Grid 响应式布局
- HTML5 Geolocation API 地理定位
- LocalStorage 本地数据持久化

### 2.2 设计风格
- **主题**: 校园清新风格
- **主色调**: 蓝色 (#4A90D9) + 绿色 (#52C41A)
- **背景色**: 浅灰白 (#F5F7FA)
- **卡片风格**: 圆角卡片 + 柔和阴影
- **字体**: 系统字体栈 (无外部依赖)

---

## 3. 功能模块

### 3.1 首页/拼单列表页
- 顶部导航栏（Logo + 筛选按钮）
- 校区筛选器（全部/东校区/西校区/南校区/北校区）
- 拼单卡片列表（按发布时间倒序）
- 底部浮动发布按钮
- 空状态提示（暂无拼单）

### 3.2 拼单卡片组件
显示信息：
- 商家名称 + 商品描述
- 截止时间（倒计时显示）
- 已参与人数 / 目标人数
- 人均费用
- 校区/宿舍楼位置
- 发起人（匿名显示）

### 3.3 发布拼单页
输入表单：
- 商家名称（必填）
- 商品描述（必填）
- 总金额（元，必填，正数）
- 目标人数（必填，2-20）
- 截止时间（必填，时间选择器）
- 校区选择（必填，下拉选择）
- 宿舍楼（可选）
- 发起人微信号（必填）
- 商品类型（外卖/奶茶/超市/其他）

处理逻辑：
- 表单验证（金额正数、人数范围、时间有效）
- 生成唯一拼单ID
- 状态设为"进行中"
- 存储到LocalStorage

### 3.4 拼单详情页/加入页
- 完整拼单信息展示
- AA费用计算结果
- 参与人数进度条
- "加入拼单"按钮
- 一键复制微信号功能（显示微信号后复制）
- 拼单状态标识（进行中/已结束）

### 3.5 拼单推荐模块
- 基于用户历史浏览/参与记录
- 推荐相同校区、相似商品类型的拼单
- 推荐列表显示在首页顶部

### 3.6 地理位置服务
- 首次使用时请求位置权限
- 通过浏览器Geolocation API获取坐标
- 根据坐标匹配最近校区
- 自动填充定位信息

### 3.7 拼单状态自动更新
- 页面加载时检查所有拼单
- 截止时间已过的拼单自动标记为"已结束"
- 已结束的拼单显示但不可加入

## 4. 数据结构

### 4.1 拼单数据 (LocalStorage: "groupbuys")
```javascript
{
  id: "GB_xxxxxxxx",
  merchant: "商家名称",
  description: "商品描述",
  totalAmount: 100.00,
  targetCount: 4,
  currentCount: 1,
  deadline: "2024-04-27T18:00",
  campus: "east", // east/west/south/north
  dormitory: "宿舍楼信息",
  wechatId: "微信号",
  category: "food", // food/drink/supermarket/other
  creatorId: "browser_id",
  status: "active", // active/ended
  createdAt: "2024-04-27T12:00:00"
}
```

### 4.2 用户数据 (LocalStorage: "user_data")
```javascript
{
  browserId: "uuid",
  history: [
    { campus: "east", categories: ["food", "drink"] }
  ],
  preferences: {
    defaultCampus: "east"
  }
}
```

## 5. 交互设计

### 5.1 按钮反馈
- 点击变色效果
- 加载状态显示（转圈动画）
- 成功/失败Toast提示

### 5.2 操作流程
- **发布拼单**: 点击→填写表单→确认发布（≤3步）
- **加入拼单**: 点击卡片→查看详情→复制微信号（≤3步）

### 5.3 动画效果
- 页面切换：淡入淡出
- 卡片加载：依次渐入
- 按钮点击：缩放反馈
- Toast提示：从下滑入/上滑收起

---

## 6. 响应式断点

| 设备 | 宽度 | 布局 |
|------|------|------|
| 手机 | < 768px | 单列卡片 |
| 平板 | 768px-1024px | 双列卡片 |
| PC | > 1024px | 三列卡片 + 侧边栏 |

---

## 7. 精度要求

- **费用计算**: 保留两位小数
- **时间显示**: 精确到分钟（倒计时显示到秒）
- **排序精度**: 发布时间精确到秒

---

## 8. 错误处理

| 故障类型 | 处理方式 |
|----------|----------|
| 网络中断 | Toast提示"网络连接失败，请检查网络" |
| 表单验证失败 | 红色边框 + 输入框下方错误提示 |
| 拼单已过期 | 自动显示"已结束"标签，禁止加入 |
| 位置获取失败 | 使用默认校区，允许手动选择 |
| LocalStorage满 | 提示清理旧数据或联系管理员 |

---

## 9. 安全要求

- 不存储用户密码
- 微信号仅展示，不加密存储
- 使用浏览器指纹生成用户ID
- 无登录系统，即开即用

---
